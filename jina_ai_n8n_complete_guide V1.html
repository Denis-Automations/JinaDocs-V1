1Ô∏è‚É£ Reader API - Web Content Extraction

r.jina.ai Output-based tokens

What It Does

  • Converts any URL to clean, LLM-friendly Markdown
  • Removes ads, navigation, and clutter
  • Extracts main content intelligently
  • Supports images, links summary, and alt text
  • Handles JavaScript-rendered pages

Use Cases

  • Blog post extraction for analysis
  • Competitor website monitoring
  • Building knowledge bases from web sources
  • Content aggregation and curation

cURL Example

curl -X POST 'https://r.jina.ai/' \
  -H "Authorization: Bearer YOUR_API_KEY" \
  -H "Content-Type: application/json" \
  -H "Accept: application/json" \
  -H "X-Return-Format: markdown" \
  -H "X-With-Links-Summary: true" \
  -H "X-With-Images-Summary: true" \
  -d '{
    "url": "https://example.com"
  }'

n8n HTTP Request Setup

{
  "method": "POST",
  "url": "https://r.jina.ai/",
  "authentication": "headerAuth",
  "headerAuth": {
    "name": "Authorization",
    "value": "Bearer YOUR_API_KEY"
  },
  "headers": {
    "Content-Type": "application/json",
    "Accept": "application/json",
    "X-Return-Format": "markdown",
    "X-With-Links-Summary": "true"
  },
  "body": {
    "url": "{{ $json.url }}"
  }
}

Key Headers

  • X-Return-Format: markdown, html, text, screenshot, pageshot
  • X-With-Links-Summary: true/all (extract all links)
  • X-With-Images-Summary: true/all (extract all images)
  • X-Target-Selector: CSS selector to focus on specific elements
  • X-Remove-Selector: CSS selector to exclude elements
  • X-No-Cache: true (bypass cache for fresh content)
  • X-Timeout: Max wait time in seconds

  1. Add Jina AI node to your workflow
  2. Click on Credentials dropdown
  3. Select Create New
  4. Paste your API key

  1. Add HTTP Request node
  2. Go to Authentication section
  3. Choose Generic Credential Type ‚Üí Header Auth
  4. Configure:
    • Name: Authorization
    • Value: Bearer YOUR_JINA_API_KEY
  5. Save credential

Token Management

Monitor Your Usage

  • Check token balance at jina.ai ‚Üí API KEY & BILLING
  • Each API response includes usage.tokens field
  • Track usage in n8n workflows using Set node

Optimize Token Usage

Reader API

  • Use CSS selectors (X-Target-Selector) to extract only needed content
  • Remove unwanted sections with X-Remove-Selector
  • Avoid ReaderLM-v2 unless necessary (costs 3x tokens)

Search API

  • Minimize number of results (num parameter)
  • Each search costs 10K tokens regardless of results
  • Cache results when possible

DeepSearch

  • Use low reasoning effort to save tokens
  • Set budget_tokens limit
  • Reserve for high-value research only

Embeddings

  • Batch multiple inputs in one request
  • Use Segmenter (FREE) to chunk before embedding
  • Choose appropriate model size

Handle Rate Limits

Add Delays in n8n

  • Between API calls: 100-200ms delay
  • After high-volume operations: 1-2 second delay
  • Use Wait node in n8n

Error Handling

Try Node:
  - HTTP Request (Jina API)
Catch Node (on error):
  - IF node (check if 429 rate limit)
    ‚Üì Yes
  - Wait node (exponential backoff)
  - Retry HTTP Request
    ‚Üì No
  - Log error
  - Send alert

Common Error Codes

Error Cause Solution
401 Invalid API key Check authorization header
429 Rate limit exceeded Add delay nodes, implement backoff
403 Commercial use on free tier Upgrade to paid tier
500 Server error Retry with exponential backoff
503 Service unavailable (VLM cold start) Retry after 30-60 seconds

Security Best Practices

  • Store API keys in n8n environment variables, never hardcode
  • Use n8n credential system for API key management
  • Regularly rotate API keys
  • Monitor usage for unusual patterns
  • Set up alerts for token threshold (e.g., 80% usage)

Performance Tips

  • Cache results: Use X-No-Cache: false when appropriate
  • Parallel processing: Use n8n's Split In Batches for multiple URLs
  • Segmenter is FREE: Always use it before embeddings
  • Reranker after search: Improves RAG accuracy significantly
  • Batch embeddings: Process multiple texts in one API call
